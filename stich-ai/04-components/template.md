# UI Component System (Stich AI)

Du bist ein Senior UI/UX Designer und Design System Architect.

Erstelle ein konsistentes, wiederverwendbares Component System für eine moderne Web- oder App-Anwendung.

---

## 1. Ziel

Alle UI-Elemente sollen modular, wiederverwendbar und konsistent sein.

Keine einmaligen Designs — alles basiert auf einem System.

---

## 2. Core Components

Definiere die wichtigsten UI-Bausteine:

- Buttons
- Cards
- Inputs / Forms
- Navigation (Sidebar / Topbar / Bottom Nav)
- Tables
- Modals
- Charts / Data Visuals
- Alerts / Notifications

---

## 3. Button System

Definiere Button Variants:

- Primary Button (Hauptaktion)
- Secondary Button
- Ghost Button
- Danger Button

Regeln:
- gleiche Höhe
- gleiche Rundung
- klare Hover States

---

## 4. Card System

Cards sind die wichtigste UI Einheit.

Definiere:

- KPI Cards (Stats)
- Content Cards
- Interactive Cards

Regeln:
- klare Padding Struktur
- leichte Schatten
- konsistente Eckenradius

---

## 5. Input System

Definiere Form Elemente:

- Text Input
- Search Input
- Dropdown
- Checkbox
- Toggle Switch

Regeln:
- klare Fokus States
- saubere Labels
- gute Lesbarkeit

---

## 6. Data Components

- Charts (Line, Bar, Pie)
- Tables (sortierbar)
- Lists

---

## 7. Feedback Components

- Success Message
- Error Message
- Warning Message
- Loading States (Skeletons)

---

## 8. Design Konsistenz Regeln

- gleiche Abstände überall
- gleiche Border Radius Werte
- gleiche Schatten Logik
- gleiche Interaktionslogik

---

## OUTPUT

Erstelle ein vollständiges Component System mit:
- Komponenten Liste
- Struktur pro Komponente
- klare UI Logik
- konsistente Design Regeln
